# Contributing to FlowPilot

Thank you for your interest in contributing to FlowPilot.

FlowPilot is an open-source project developed by Nura with the goal of advancing serious, policy-driven authorization models for agentic AI systems.

We welcome contributions that improve correctness, clarity, security, and governance.

---

## Guiding Principles

- Authorization is security-critical infrastructure
- Explicit is better than implicit
- Governance matters as much as enforcement
- Changes must be auditable and reversible

---

## How to Contribute

1. Fork the repository
2. Create a feature branch (`feat/...`, `fix/...`, `docs/...`)
3. Make focused, well-documented changes
4. Add or update tests where applicable
5. Open a pull request with a clear rationale

---

## Development Setup

- Python 3.11+
- Docker (for full-stack testing)
- uv recommended for Python environments
- Pre-commit hooks encouraged

---

## Code Standards

- Clear, explicit naming
- No hidden side effects
- No embedded policy logic in application code
- Security and privacy assumptions documented

---

## Security

Do not include secrets or real PII in code, tests, or issues.

For security concerns, contact: security@nura.pro

---
